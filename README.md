# Анализ пользовательского трафика сайта

## 1. Цель проекта

Цель проекта — проанализировать поведение пользователей на сайте и выявить ключевые факторы, влияющие на конверсию и качество трафика.

В рамках анализа решаются следующие задачи:

- Определение источников трафика с наибольшей долей пользователей с целевыми действиями  
- Анализ воронки поведения и выявление этапов максимального оттока пользователей  
- Оценка влияния факторов (устройство, источник, время) на качество трафика  
- Формирование рекомендаций по повышению конверсии и эффективности привлечения пользователей  

---

## 2. Данные

Используется датасет `website_traffic.csv`, содержащий 1744 записи и 24 признака.

Данные представляют собой лог пользовательских событий на сайте.

### Основные группы признаков:

**Идентификаторы:**
- id
- client_id
- session_id

**Временные признаки:**
- created_at
- client_timestamp

**События и поведение:**
- event_type
- page_path
- pages_viewed
- time_on_site
- scroll_depth

**Маркетинговые параметры:**
- utm_source
- utm_medium
- utm_campaign
- utm_content

**Технические параметры:**
- device_type
- os
- browser

## 3. Этапы анализа

### 1. Подготовка и очистка данных
- обработка пропусков  
- удаление дубликатов  
- фильтрация нерелевантных событий  

### 2. Исследовательский анализ данных (EDA)
- распределение трафика  
- анализ источников  
- базовые метрики поведения пользователей  

### 3. Поведенческий анализ
- построение пользовательской воронки  
- анализ drop-off на каждом этапе  
- сегментация пользователей по качеству  

### 4. Анализ сегментов
- устройство (mobile / desktop / tablet)  
- время активности  
- источники трафика  

### 5. Выводы и рекомендации
- ключевые проблемы продукта  
- точки роста конверсии  
- приоритетные направления оптимизации  

---

## 4. Результаты проекта

В результате анализа были выявлены:

- основные источники качественного и некачественного трафика  
- ключевые точки потери пользователей в воронке  
- различия в поведении пользователей по устройствам  
- влияние времени и источников на качество сессий  

На основе анализа сформированы рекомендации по:
- оптимизации рекламных каналов  
- улучшению лендинга  
- повышению эффективности мобильного трафика

## 7. Примечание по запуску

Проект реализован в двух вариантах:

### SQL-версия
Изначально данные обрабатывались через PostgreSQL с использованием SQL-запросов и Python (SQLAlchemy).

Для запуска требуется локальная база данных PostgreSQL и SQL-скрипты из папки `SQL/`.

### Упрощённая версия (рекомендуется)
Для воспроизводимости проекта данные также представлены в формате CSV/JSON, что позволяет запускать анализ без базы данных.
